# Roadmap

## Fase 1 - Fondamenta

- MCP server Python minimale
- tool sicuri base
- registry agenti
- router task rule-based
- documentazione standard
- entrypoint web canonica `backend.web.app:app`
- visione AIOS formalizzata

## Fase 2 - Orchestrazione

- execution plan strutturato
- validazione pre-run
- output JSON standard per agenti
- logging decisionale
- Router con provider, modello, MCP e tool
- Supervisor con priorita, rischio e parallelizzazione

## Fase 3 - GitHub workflow

- lettura issue/PR
- generazione branch operativi
- proposta patch controllata
- apertura PR con checklist
- integrazione DevOps per CI, Docker, VPS e rollback

## Fase 4 - Voice front-end

- webhook/orchestratore per comandi vocali
- routing task da prompt parlato
- sintesi risultato finale
- endpoint Google Home, IFTTT e Gemini consolidati
- risposta adatta a sintesi vocale

## Fase 5 - Hardening

- policy sicurezza tool
- sandbox locale
- audit log
- test e quality gate
- mantenimento di una sola entrypoint canonica per componente
- permessi MCP per agente
- memoria persistente e tracciamento workflow

## Fase 6 - AIOS completo

- provider multipli dietro policy del Router
- MCP Browser, GitHub, Google Drive, Calendar, Email, Docker e SSH
- esecuzione controllata dei Flight Plan
- report finale unificato su API, web e voce
